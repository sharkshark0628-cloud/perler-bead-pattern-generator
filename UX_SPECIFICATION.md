## 1. UX Goals

目标：帮助拼豆用户用尽可能少的操作，把普通图片转换为可以直接照着制作的拼豆图纸。

> 上传图片 → 设置少量必要参数 → AI 生成三个方案 → 选择并轻量调整 →
> 获得最终图纸。

产品是**智能拼豆图纸生成器**，不是拼豆编辑器。

### UX Principles

1.  **AI First**：AI 承担主体识别、构图、背景处理和颜色匹配。
2.  **Minimal Editing**：用户仅选择方案、拖动、缩放、90° 旋转和重置。
3.  **Value Before
    Login**：首次生成无需登录；保存、导出和云端图纸需要微信登录。
4.  **Recover, Don't Restart**：从最近有效状态恢复。
5.  **Cute but Productive**：可爱、简约、柔和、有手工温度。
6.  **Pattern Data as Source of Truth**：最终 Pattern Data
    是展示、保存和导出的唯一事实来源。
7.  **What You See Is What You Make**：最终预览尽可能对应实际制作结果。

## 3. Core User Flows

### 3.1 Generate Pattern

`首页 → 选择图片 → 图片校验与质量引导 → 参数设置 → AI Loading → 三方案统一展示 → 选择方案 → 构图调整 → Final Rendering → 最终图纸`

### 3.2 Save Pattern

`最终图纸 → 保存 → 未登录则微信登录 → 自动继续 → Naming Bottom Sheet → 确认名称 → 保存 → Toast`

默认名称为"我的拼豆图纸"；用户可修改，建议最大 30 字符；空名称禁止保存。

### 3.3 Export Pattern

`最终图纸/图纸详情 → PNG 或 PDF → 未登录则微信登录 → 自动继续导出 → 成功/失败反馈`

### 3.4 View Saved Patterns

`我的图纸 → 登录状态检查 → Pattern Grid → Pattern Detail → 再次导出`

保存后的图纸不能继续编辑、恢复调整或重新生成。

## 4. View Specifications

### 4.1 首页

Hero 建立品牌感；上传卡片是唯一主
CTA；教程为次级内容。首页不承担社区或图库浏览。

### 4.2 上传图片

支持
`JPG / JPEG / PNG / WEBP`，一次一张。其他格式立即提示并要求重新选择。系统自动创建最长边
2048 px 的处理副本。

#### Smart Image Guide

上传前提示：主体清晰、光线充足、背景简单、分辨率较高。上传后可进行轻量基础质量检测。一般风险只提醒；损坏、不可解码或明显无法处理时要求重新选择。

### 4.3 参数设置

用户只控制：

-   豆板尺寸：`52×52 / 78×78 / 104×104`
-   品牌：MVP 为 `MARD`
-   制作难度：`简单 / 标准 / 精细`

默认：`78×78 / MARD / 标准`。不展示预计颜色数、预计豆量或预计生成结果。

### 4.4 AI Loading

独立 Loading 页面。三个方案全部结束后统一进入结果页，不渐进展示半成品。

非技术化步骤：

1.  分析图片
2.  智能构图
3.  颜色匹配

使用品牌化拼豆生成动画，不使用普通全屏 Spinner。

### 4.5 三方案选择

固定三个槽位：

-   完整构图
-   主体突出
-   纯净背景

默认突出推荐方案。单方案失败仍保留槽位；其他成功方案可选。全部失败进入独立失败页。

#### Press & Preview

点击更新当前选择。长按成功方案时 Hero
临时预览；松手恢复当前选择。长按不修改最终选择。

### 4.6 构图调整

支持单指拖动、双指缩放、90°
步进旋转、重置。必须保持安全边距和整数网格规则。MVP 不加入 Smart Snap。

缩放时实时轻量预览；手势结束后重新生成整数网格拼豆效果。点击"确认生成"后才生成最终
Pattern Data。

### 4.7 最终图纸

第一视觉为完整豆板。格子采用"实际颜色背景 + 简短图纸编号"，下方映射真实
MARD 色号。

展示完整图纸、颜色图例、颜色数、豆量、豆板尺寸、制作难度、保存、PNG
导出、PDF 导出。

页面只读；双指缩放只改变 View Scale。不加入 Build Mode。

### 4.8 我的图纸

双列 Pattern Grid，定位为作品收藏册。Pattern Card
展示完整豆板缩略图、`patternName`、豆板尺寸、颜色数、保存日期。

MVP 不支持搜索、筛选、批量管理、长按菜单或再次重命名。

### 4.9 我的

包含微信登录状态、我的图纸入口、设置、使用帮助、关于和退出登录。

不加入会员、积分、粉丝、关注、动态、等级、签到或社区账户体系。

## 5. Interaction Patterns

### Selection

参数卡和方案卡点击后立即切换。

### Loading

-   AI 生成：Branded Generation Animation
-   图纸列表：Skeleton
-   保存/导出/登录：Inline Button Loading
-   最终生成：Short Final Rendering

### Login Gate

首次生成不登录。保存或导出时未登录：`原操作 → 微信登录 → 登录成功 → 自动继续原操作`。失败或取消不得丢失当前图纸。

### Naming Bottom Sheet

保存前弹出。默认"我的拼豆图纸"。键盘出现时 Input 和 Primary CTA
必须可见。

### Regeneration

每个已登录用户每天允许重新生成
`5 次/天`。重新生成前确认消耗一次机会并显示剩余次数。不得使用"第一次点击重新生成时触发登录"的规则。

### Gestures

-   Tap：选择/执行
-   Long Press：仅 Press & Preview
-   One-finger Drag：移动构图
-   Two-finger Pinch：缩放
-   Rotation：按钮 90° 步进

## 6. Error & Recovery

原则：**Recover from the nearest valid state.**

-   上传失败：停留上传页，重新选择。
-   单方案失败：保留三个槽位，成功方案可选。
-   全部生成失败：独立失败页，提供"重新生成"和"重新上传图片"。
-   超时：目标 `P95 < 15 秒`；30 秒提示生成较慢并允许继续等待。
-   导出失败：停留图纸页并重试，不重新生成。
-   登录失败：保留当前 Pattern 和页面状态。
-   图纸列表失败：保留 Header/Tab Bar，仅内容区错误和重试。

## 7. MVP UX Enhancements

### UX Enhancement 001 --- Smart Image Guide

通过上传前建议和上传后轻量质量反馈降低无效输入和预期偏差。

### UX Enhancement 002 --- Press & Preview

通过长按临时预览降低三方案比较成本，不改变当前选择。

## 8. Accessibility

-   最小点击区域 `48×48 px`
-   主 CTA 高 `56 px`
-   状态不得只依赖颜色
-   动态状态表达"已选择 / 预览中 / 生成失败 / 加载中"
-   Pattern Card 使用单一可访问焦点
-   主要流程支持单手操作
-   固定 CTA 考虑安全区域

------------------------------------------------------------------------

## 9. UX Success Metrics

-   生成性能：`P95 < 15 秒`
-   生成结果满意度：`70/100`
-   轻调整后完成率：`70%`
-   保存相关目标：`50/100`

埋点口径应在 Analytics Specification 中单独定义。

------------------------------------------------------------------------

## 10. UX Scope Boundary

MVP 不包含像素级编辑、单格改色、画笔/橡皮擦、自由旋转、Smart Snap、Build
Mode、图纸再次编辑、AI 语义自动命名、社区、打赏、AI
二次创作、会员或积分体系。

Coding Agent 不得根据"常见产品功能"自行加入这些能力。
