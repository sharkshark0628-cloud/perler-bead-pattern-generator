# 1. 产品概述

## 1.1 产品简介

智能拼豆图纸生成器是一款面向拼豆爱好者的微信小程序。

产品通过图片主体理解、豆板空间感知、图像像素化、颜色量化及真实拼豆品牌色板映射，将用户上传的普通图片自动转换为可以直接用于实际拼豆制作的图纸。

用户无需掌握像素画设计、图片处理或人工配色能力，只需要：

1.  上传图片；
2.  选择豆板尺寸；
3.  选择拼豆品牌；
4.  选择制作难度；
5.  从系统生成的三个方案中选择一个；
6.  进行简单构图调整；
7.  获得可直接制作的拼豆图纸。

MVP 的核心定位为：

> **智能拼豆图纸生成器，而不是拼豆编辑器。**

核心产品原则：

> **Less manual editing, better first generation.**

内部产品理念：

> **少修图，直接拼。**

# 2. 产品背景与问题定义

拼豆用户通常通过社交媒体、拼豆社群或在线图库寻找图纸。当用户希望将个人照片、动漫图片、宠物照片或其他自定义图片制作成拼豆作品时，通常面临：

-   无法找到对应拼豆图纸；
-   不具备像素画设计能力；
-   不知道如何将普通图片转换为拼豆图；
-   不熟悉真实拼豆品牌色号；
-   难以判断图片颜色对应哪个实际拼豆颜色；
-   手工统计每种豆子数量耗时；
-   普通图片直接像素化后主体可能过小；
-   复杂背景占用有限豆板空间；
-   自动生成图纸容易出现颜色碎片化；
-   用户需要大量逐像素修改才能实际制作。

核心问题：

> 用户希望使用拼豆还原自己喜欢的图片，但无法快速获得适合真实豆板和实际拼豆品牌的可制作图纸。

本产品解决的不是"如何把图片变成像素画"，而是：

> **如何理解图片内容，并结合真实拼豆制作约束，自动生成更适合实际制作的拼豆图纸？**

# 7. 生成参数

## 7.1 豆板尺寸

MVP 仅支持：

-   52×52
-   78×78
-   104×104

不支持自定义尺寸。

## 7.2 拼豆品牌

MVP 仅支持：

> **MARD**

使用人工核对的 **MARD Reference Palette**。

除非获得官方数据或授权，产品不得宣称使用 "Official MARD Palette"。

## 7.3 制作难度

支持：

-   **简约**：较少颜色，降低颜色碎片和制作难度；
-   **标准**：平衡还原度与制作难度，默认选项；
-   **精细**：使用更多颜色，尽可能保留细节。

具体目标颜色数量不得简单固定为
8/16/32。系统应结合可用图案区域、图片复杂度和制作难度动态确定颜色复杂度。

# 8. 核心业务规则

## BR-001 豆板安全边距

有效拼豆图案与豆板上、下、左、右边界之间均必须至少保留 **5 格空白**。

  豆板尺寸   最大可用图案区域
  ---------- ------------------
  52×52      42×42
  78×78      68×68
  104×104    94×94

所有构图调整必须受安全区域硬边界限制。

# 9. 图片上传需求

## US-001 图片上传

用户一次只能上传一张图片。

支持：

-   JPG
-   JPEG
-   PNG
-   WEBP

暂不支持：

-   GIF
-   HEIC 原始文件
-   动态图片
-   视频

图片选择后立即校验实际文件类型，不得仅检查扩展名。

不支持时提示：

> 无法读取这张图片，请更换 JPG、PNG 或 WEBP 图片后重试。

## 9.1 高清图片处理

系统自动创建最长边为 **2048 px** 的工作副本。用户无需手动压缩。

## 9.2 低清图片

低清晰度属于 Warning，不阻断生成。

提示：

> 图片清晰度较低，生成图纸可能丢失部分细节。

# 10. 三方案智能生成

## US-004 三方案生成

每轮必须同时执行：

1.  完整图；
2.  主体突出；
3.  去背景。

页面始终展示三个固定方案槽位。

### 完整图

尽可能保留原始图片整体构图和主要视觉信息。

### 主体突出

识别主要主体，通过自动裁剪和缩放提高主体在可用图案区域中的占比。

### 去背景

识别主要前景主体，执行前景分割并移除背景。

## 10.1 多主体图片

MVP 不提供手动选择主体。

系统根据视觉显著性、主体空间关系和组合关系自动处理。多个主体构成视觉组合时，应尽可能保留主要主体组。

## 10.2 单方案失败

单个策略失败不得导致整轮失败。

失败槽位显示：

> 未能识别明确主体

或：

> 该方案生成失败

其他成功方案仍可选择。失败方案不可选择。

------------------------------------------------------------------------

# 11. 生成性能与超时

0--15 秒显示正常生成状态。

超过 15 秒提示：

> 生成时间比预计稍长，请继续等待。

达到 30 秒，本轮生成超时。

若已有方案成功，则展示成功方案，未完成槽位标记"生成超时"。

若三个方案均未完成，则本轮生成失败，并允许系统失败重试。

系统失败重试不消耗主动重新生成额度。

------------------------------------------------------------------------

# 12. 重新生成

用户可以主动重新生成三个候选方案。

每日限制：

> **5 次/天**

重新生成不要求登录。

MVP
使用微信小程序本地存储记录日期和当日重新生成次数。该限制属于轻量使用限制，不是严格反滥用机制。

以下情况不扣除额度：

-   网络失败；
-   服务器失败；
-   三个方案全部失败。

至少一个新方案成功后扣除 1 次。

达到上限后禁用重新生成按钮，并提示：

> 今日重新生成次数已用完，明天再来试试吧。

------------------------------------------------------------------------

# 13. 方案选择

用户必须选择一个成功方案。

选中方案显示明确选中状态。

点击"下一步"进入构图调整页面。

# 14. 构图调整

## US-006 构图调整

支持：

-   单指拖动；
-   双指缩放；
-   90°步进旋转；
-   重置。

## 14.1 移动

单指拖动图案。采用硬边界限制，不得突破 5 格安全边距。

## 14.2 缩放

双指缩放。

最终有效图案必须映射到整数拼豆网格，不得出现半格拼豆。

手势过程中显示轻量实时预览。

用户停止缩放后，执行整数网格重采样和精确颜色映射，目标 **\<1 秒**
更新精确拼豆预览。

## 14.3 旋转

仅支持：

-   0°
-   90°
-   180°
-   270°

不支持自由角度旋转。

视觉反馈目标 \<300 ms。

## 14.4 重置

恢复进入构图调整页面时的初始状态。

# 15. 最终拼豆图纸

最终输出定义为：

> **Craft-ready Bead Pattern**

用户无需再次裁剪、计算豆板位置、判断颜色或统计豆量，可以根据完整豆板坐标网格和图纸编号逐格摆放拼豆。

## 15.1 完整豆板

最终图纸必须展示用户选择的完整豆板网格。

即使图案只占部分区域，也必须显示完整的 52×52、78×78 或 104×104 豆板。

## 15.2 坐标

显示横向和纵向坐标，坐标从 1 开始。

## 15.3 网格

每一个格子对应 1 颗拼豆。

空白格表示不放拼豆。

## 15.4 格子显示

有效拼豆格显示：

> **实际颜色背景 + 简短图纸编号**

格子中不直接显示完整 MARD 色号。

------------------------------------------------------------------------

# 16. 色号映射与颜色匹配

图纸编号映射到 MARD Reference Palette 色号。

示例：

  图纸编号   MARD 色号   颜色名称     数量
  ---------- ----------- ---------- ------
  ①          A01         白色          126
  ②          B12         黄色           84
  ③          G07         深灰           51

颜色匹配目标采用：

> **CIELAB + CIEDE2000**

系统在感知颜色空间中计算输入颜色与品牌色板之间的视觉色差，并选择感知上最接近的可用品牌颜色。

不使用简单 RGB 欧氏距离作为最终颜色匹配标准。

------------------------------------------------------------------------

# 17. 豆量统计

系统自动统计：

-   每种色号数量；
-   使用颜色总数；
-   拼豆总数。

强验收条件：

> **所有颜色数量之和 = 所有非空网格数量**

------------------------------------------------------------------------

# 18. 图纸查看

最终图纸支持：

-   双指缩放查看；
-   单指拖动查看局部区域。

这里的缩放仅改变 View Scale，不得修改 Pattern Data。

# 19. 登录与权限

产品采用 Value-first Authentication。

原则：

> **查看价值无需登录，保存价值需要登录。**

## 19.1 无需登录

-   上传图片；
-   首次生成；
-   查看三个方案；
-   重新生成；
-   选择方案；
-   构图调整；
-   生成最终图纸；
-   查看最终图纸。

## 19.2 需要微信登录

-   导出图片；
-   导出 PDF；
-   保存到"我的图纸"。

## 19.3 登录状态恢复

用户因保存或导出触发登录后，登录成功必须自动继续原操作。

登录取消或失败时：

-   当前最终图纸不得丢失；
-   用户停留在最终图纸页；
-   提示"登录后才能保存图纸"。

------------------------------------------------------------------------

# 20. 图片与 PDF 导出

## 20.1 图片导出

已登录用户可以导出图片格式图纸。

失败提示：

> 导出失败，请重试。

重新导出不得重新执行主体识别、像素化或颜色匹配。

## 20.2 PDF 导出

PDF 应包含：

-   完整豆板图纸；
-   坐标；
-   图纸编号；
-   MARD 色号表；
-   每种豆子数量；
-   总颜色数；
-   总豆数。

Pattern Generation 与 Pattern Export 必须逻辑分离。

------------------------------------------------------------------------

# 21. 我的图纸

"我的图纸"是最终图纸历史库，不是可编辑项目空间。

## 21.1 保存内容

保存：

-   最终 Pattern Data；
-   豆板尺寸；
-   品牌；
-   制作难度；
-   色号统计；
-   豆量统计；
-   创建时间；
-   必要导出信息。

不保存：

-   原始图片；
-   2048 px 工作副本；
-   三个候选方案；
-   主体识别结果；
-   构图调整历史。

## 21.2 用户操作

用户以后可以：

-   查看；
-   导出图片；
-   导出 PDF。

不能返回构图调整页面继续修改。

## 21.3 保存幂等性

每张最终图纸获得唯一 `pattern_id`。

相同 `pattern_id` 重复保存不得创建重复记录。

# 23. 隐私要求

原始图片和处理工作副本默认仅用于当前生成会话。

不得默认：

-   长期保存；
-   用于模型训练；
-   用于算法训练集；
-   用于产品宣传。

未来若使用用户图片改善算法，必须单独取得明确授权，不得将授权隐藏在"保存图纸"操作中。

------------------------------------------------------------------------

# 24. 安全要求

-   微信用户身份必须由后端验证；
-   后端不得信任客户端直接提交的 user_id；
-   用户只能读取自己的已保存图纸；
-   用户 A 不得通过修改 pattern_id 读取用户 B 的图纸；
-   上传图片必须验证实际文件类型；
-   导出文件必须基于系统认可的最终 Pattern Data。

# 31. Product Risks

## R-001 首次生成质量风险

产品核心差异化依赖"完整图 / 主体突出 /
去背景"三个方案提高首次生成可用率。

核心指标：

> SM-002 ≥ 70%

若 Beta 阶段 SM-002 \< 40%，应优先改善：

-   主体识别；
-   图像分割；
-   构图策略；
-   图像缩放；
-   颜色碎片控制；
-   方案多样性。

不得优先投入社区、图库、库存或更多品牌。
